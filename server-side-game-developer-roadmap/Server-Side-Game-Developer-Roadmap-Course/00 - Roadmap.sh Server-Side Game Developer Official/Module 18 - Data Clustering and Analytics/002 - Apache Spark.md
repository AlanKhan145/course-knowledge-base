## 1. Tóm tắt
Nội dung này tập trung vào **Apache Spark** trong lộ trình Server-Side Game Developer. Sau phần này, bạn nên hiểu vai trò của chủ đề trong game backend và tạo được một artifact kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Dùng **Apache Spark** để hiểu hành vi người chơi, retention, anti-cheat hoặc matchmaking thông minh.
- Biết khác nhau giữa batch analytics, streaming events và online inference.
- Tạo mini pipeline hoặc design note cho telemetry/ML use case.

## 3. Nội dung roadmap
- Batch processing
- Distributed data processing
- Player telemetry
- Economy analytics
- Fraud/cheat signal processing

## 4. Bài tập thực hành
- Chọn một telemetry event và mô tả pipeline lưu/xử lý/phân tích.
- Tạo cohort hoặc segmentation idea cho retention/matchmaking.
- Nêu cách tránh dùng ML quá sớm khi rule-based system đủ tốt.

## 5. Artifact nên tạo
- Telemetry pipeline
- Segmentation note
- ML use-case brief

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Apache Spark** trong kiến trúc game backend không?
- Chủ đề này ảnh hưởng đến latency, reliability, scaling, consistency, security hay observability?
- Artifact nào từ bài này có thể đưa vào portfolio server-side game developer?

## 7. Tổng kết
**Apache Spark** nên được học bằng cách thiết kế, code thử, đo đạc và ghi lại trade-off. Hãy biến bài học thành một demo, diagram hoặc README có thể review.
