# 005 - Erlang

**Module:** Module 07 - Programming Languages cho Game Server
**Roadmap item:** Bài 25
**Loại nội dung:** Server programming language
**Thứ tự trong module:** 005
**Thời lượng gợi ý:** 25-35 phút

---

## 1. Tóm tắt
Nội dung này tập trung vào **Erlang** trong lộ trình Server-Side Game Developer. Sau phần này, bạn nên hiểu vai trò của chủ đề trong game backend và tạo được một artifact kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Biết **Erlang** phù hợp với loại game server hoặc backend service nào.
- Nhận biết thế mạnh về concurrency, performance, ecosystem và tooling.
- Tạo bài tập nhỏ về network service, room, matchmaking hoặc worker.

## 3. Nội dung roadmap
- Fault tolerance
- Actor/concurrent system
- Dùng cho hệ thống chat, realtime, telecom-style backend

## 4. Bài tập thực hành
- Viết service nhỏ nhận request/message và trả response bằng ngôn ngữ đang học.
- Ghi lại cách ngôn ngữ đó xử lý concurrency và error handling.
- So sánh với một ngôn ngữ khác trong roadmap.

## 5. Artifact nên tạo
- Mini service
- Language trade-off note
- Concurrency example

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Erlang** trong kiến trúc game backend không?
- Chủ đề này ảnh hưởng đến latency, reliability, scaling, consistency, security hay observability?
- Artifact nào từ bài này có thể đưa vào portfolio server-side game developer?

## 7. Tổng kết
**Erlang** nên được học bằng cách thiết kế, code thử, đo đạc và ghi lại trade-off. Hãy biến bài học thành một demo, diagram hoặc README có thể review.
