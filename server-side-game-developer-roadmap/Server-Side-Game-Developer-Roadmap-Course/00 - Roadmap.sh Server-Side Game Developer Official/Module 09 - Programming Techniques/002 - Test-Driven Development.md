# 002 - Test-Driven Development

**Module:** Module 09 - Programming Techniques
**Roadmap item:** Bài 32
**Loại nội dung:** Programming technique
**Thứ tự trong module:** 002
**Thời lượng gợi ý:** 25-35 phút

---

## 1. Tóm tắt
Nội dung này tập trung vào **Test-Driven Development** trong lộ trình Server-Side Game Developer. Sau phần này, bạn nên hiểu vai trò của chủ đề trong game backend và tạo được một artifact kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Áp dụng **Test-Driven Development** để game server dễ test, debug và bảo trì hơn.
- Biết giảm coupling giữa game logic, storage, cache, queue và transport.
- Tạo code note hoặc checklist kỹ thuật cho production issue.

## 3. Nội dung roadmap
- Unit test game logic
- Integration test API
- Load test server
- Mock network/database

## 4. Bài tập thực hành
- Refactor một đoạn game logic giả lập để dễ test hơn.
- Viết unit test hoặc test case cho rule quan trọng.
- Ghi lại bug class mà kỹ thuật này giúp phòng tránh.

## 5. Artifact nên tạo
- Test cases
- Refactor note
- Debug checklist

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Test-Driven Development** trong kiến trúc game backend không?
- Chủ đề này ảnh hưởng đến latency, reliability, scaling, consistency, security hay observability?
- Artifact nào từ bài này có thể đưa vào portfolio server-side game developer?

## 7. Tổng kết
**Test-Driven Development** nên được học bằng cách thiết kế, code thử, đo đạc và ghi lại trade-off. Hãy biến bài học thành một demo, diagram hoặc README có thể review.
