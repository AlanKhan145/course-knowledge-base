# 001 - Database Fundamentals

**Module:** Module 13 - Database cho Game Backend
**Roadmap item:** Bài 48
**Loại nội dung:** Game data storage
**Thứ tự trong module:** 001
**Thời lượng gợi ý:** 25-35 phút

---

## 1. Tóm tắt
Nội dung này tập trung vào **Database Fundamentals** trong lộ trình Server-Side Game Developer. Sau phần này, bạn nên hiểu vai trò của chủ đề trong game backend và tạo được một artifact kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Dùng **Database Fundamentals** để lưu account, inventory, profile, leaderboard hoặc telemetry đúng nhu cầu.
- Biết trade-off consistency, latency, indexing, cache và migration.
- Tạo schema/cache/query note cho một service game backend.

## 3. Nội dung roadmap
- Persistent data
- Transaction
- Index
- Query
- Consistency
- Latency

## 4. Bài tập thực hành
- Thiết kế bảng/collection/cache key cho profile, inventory hoặc leaderboard.
- Ghi query chính, index cần có và policy cache invalidation.
- Nêu cách backup, migration hoặc consistency khi có giao dịch vật phẩm.

## 5. Artifact nên tạo
- Schema/cache design
- Query/index note
- Data consistency checklist

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Database Fundamentals** trong kiến trúc game backend không?
- Chủ đề này ảnh hưởng đến latency, reliability, scaling, consistency, security hay observability?
- Artifact nào từ bài này có thể đưa vào portfolio server-side game developer?

## 7. Tổng kết
**Database Fundamentals** nên được học bằng cách thiết kế, code thử, đo đạc và ghi lại trade-off. Hãy biến bài học thành một demo, diagram hoặc README có thể review.
