# 001 - Server-side game là gì?

**Module:** Module 01 - Tổng quan Server-Side Game Development

**Loại nội dung:** Server-side game foundation
**Thứ tự trong module:** 001
**Thời lượng gợi ý:** 25-35 phút

---

## 1. Tóm tắt
Nội dung này tập trung vào **Server-side game là gì?** trong lộ trình Server-Side Game Developer. Sau phần này, bạn nên hiểu vai trò của chủ đề trong game backend và tạo được một artifact kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Giải thích được **Server-side game là gì?** trong bối cảnh server-side game development.
- Biết chủ đề này nằm ở đâu trong game backend architecture.
- Tạo được ghi chú hoặc demo nhỏ để đưa vào portfolio.

## 3. Nội dung roadmap
- Vai trò của server trong game online/multiplayer
- Phân biệt client-side và server-side game development
- Server xử lý gì: đăng nhập, matchmaking, room, combat, inventory, leaderboard, chat, persistence
- Kiến trúc cơ bản: Client → Gateway/API → Game Server → Database/Cache/Queue

## 4. Bài tập thực hành
- Tóm tắt bài học trong 5 dòng bằng lời của bạn.
- Tạo một demo hoặc architecture note nhỏ.
- Liên hệ bài học với một capstone project cuối khóa.

## 5. Artifact nên tạo
- Study note
- Prototype
- Portfolio README

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Server-side game là gì?** trong kiến trúc game backend không?
- Chủ đề này ảnh hưởng đến latency, reliability, scaling, consistency, security hay observability?
- Artifact nào từ bài này có thể đưa vào portfolio server-side game developer?

## 7. Tổng kết
**Server-side game là gì?** nên được học bằng cách thiết kế, code thử, đo đạc và ghi lại trade-off. Hãy biến bài học thành một demo, diagram hoặc README có thể review.
