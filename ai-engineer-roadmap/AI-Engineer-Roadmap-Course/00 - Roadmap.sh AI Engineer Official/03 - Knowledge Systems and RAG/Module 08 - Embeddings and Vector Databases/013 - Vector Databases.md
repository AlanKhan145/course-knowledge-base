## 3. Khai niem chinh

- Vector databases store embeddings and find nearest neighbors by similarity.
- They are useful when keyword search is not enough and semantic similarity matters.
- Important design choices include metadata filters, index type, top-k, recall, latency and update strategy.

## 4. Vi du / Demo

```text
text -> chunk -> embedding -> vector DB -> query vector -> top-k results
```
