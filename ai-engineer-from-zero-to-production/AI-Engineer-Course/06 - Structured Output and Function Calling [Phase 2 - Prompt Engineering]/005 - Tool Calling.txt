# Tool Calling

## Overview
Tool Calling is the broader pattern that encompasses Function Calling — it refers to giving a model access to a set of named tools (functions, APIs, search engines, code interpreters) and letting it orchestrate calls to those tools across multiple steps. This lesson focuses on multi-tool scenarios, parallel tool execution, tool design best practices, and the differences in tool calling across providers (OpenAI, Anthropic, and framework abstractions like LangChain).

## Key Points
- Tool Calling and Function Calling are often used interchangeably in OpenAI's API; the distinction is that "tools" is the more general concept (a tool can be a function, a code interpreter, or a retrieval system)
- Parallel tool calling (introduced in GPT-4 Turbo) allows the model to emit multiple `tool_calls` in a single response — execute them concurrently using `asyncio` or `ThreadPoolExecutor` for significant latency savings
- Tool design follows the single-responsibility principle: one tool does one thing well; a tool named `do_everything` defeats the model's ability to reason about when to use it
- Anthropic Claude uses a `tools` array with the same concept but different field names (`input_schema` instead of `parameters`); abstract provider differences behind an adapter layer in production
- Tool descriptions should explain WHEN to use the tool (preconditions), not just WHAT it does — this dramatically reduces unnecessary or misapplied tool calls
- The `tool_choice` parameter controls model autonomy: `"none"` disables tools, `"auto"` lets the model decide, `"required"` forces at least one tool use
- Always implement a maximum iteration guard in your tool-calling loop; without it, a misbehaving model can get into an infinite call cycle

## Code Example
```python
# pip install openai
import json
import asyncio
from concurrent.futures import ThreadPoolExecutor
from openai import OpenAI

client = OpenAI()

# ── Tool definitions ─────────────────────────────────────────────────────────

tools = [
    {
        "type": "function",
        "function": {
            "name": "search_web",
            "description": "Search the web for current information on a topic. Use when the user asks about recent events or facts not in your training data.",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "The search query string"}
                },
                "required": ["query"],
                "additionalProperties": False
            },
            "strict": True
        }
    },
    {
        "type": "function",
        "function": {
            "name": "calculate",
            "description": "Evaluate a safe mathematical expression and return the numeric result. Use for arithmetic, percentages, and simple algebra.",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {"type": "string", "description": "A safe math expression, e.g. '(120 * 0.15) + 50'"}
                },
                "required": ["expression"],
                "additionalProperties": False
            },
            "strict": True
        }
    },
    {
        "type": "function",
        "function": {
            "name": "get_exchange_rate",
            "description": "Get the current exchange rate between two currencies. Use when the user needs currency conversion.",
            "parameters": {
                "type": "object",
                "properties": {
                    "from_currency": {"type": "string", "description": "ISO 4217 code, e.g. 'USD'"},
                    "to_currency": {"type": "string", "description": "ISO 4217 code, e.g. 'EUR'"}
                },
                "required": ["from_currency", "to_currency"],
                "additionalProperties": False
            },
            "strict": True
        }
    }
]

# ── Simulated tool implementations ───────────────────────────────────────────

def search_web(query: str) -> str:
    return f"[Search result for '{query}']: The latest AI models were released in Q3 2024."

def calculate(expression: str) -> str:
    try:
        # In production: use a safe math parser, NOT eval()
        result = eval(expression, {"__builtins__": {}}, {})  # noqa: S307
        return str(result)
    except Exception as e:
        return f"Error: {e}"

def get_exchange_rate(from_currency: str, to_currency: str) -> str:
    rates = {"USD_EUR": 0.92, "USD_GBP": 0.79, "EUR_USD": 1.09}
    key = f"{from_currency}_{to_currency}"
    rate = rates.get(key, 1.0)
    return json.dumps({"rate": rate, "from": from_currency, "to": to_currency})

TOOL_MAP = {
    "search_web": search_web,
    "calculate": calculate,
    "get_exchange_rate": get_exchange_rate
}

# ── Execute tool calls in parallel using a thread pool ───────────────────────

def execute_tool_calls_parallel(tool_calls) -> list[dict]:
    """Run all tool calls concurrently and return tool result messages."""
    def run_one(tc):
        fn_name = tc.function.name
        fn_args = json.loads(tc.function.arguments)
        result = TOOL_MAP[fn_name](**fn_args)
        return {
            "role": "tool",
            "tool_call_id": tc.id,
            "content": result
        }

    with ThreadPoolExecutor() as executor:
        results = list(executor.map(run_one, tool_calls))
    return results

# ── Agentic loop with parallel execution and iteration cap ───────────────────

def run_tool_agent(user_message: str, max_iterations: int = 8):
    messages = [
        {"role": "system", "content": "You are a helpful financial and research assistant."},
        {"role": "user", "content": user_message}
    ]

    for iteration in range(max_iterations):
        response = client.chat.completions.create(
            model="gpt-4o",
            tools=tools,
            tool_choice="auto",
            messages=messages
        )
        assistant_msg = response.choices[0].message
        messages.append(assistant_msg)

        if not assistant_msg.tool_calls:
            print(f"Final answer (after {iteration} tool iterations):")
            print(assistant_msg.content)
            return

        print(f"Iteration {iteration + 1}: {len(assistant_msg.tool_calls)} tool call(s)")
        tool_results = execute_tool_calls_parallel(assistant_msg.tool_calls)
        messages.extend(tool_results)

    print("WARNING: Reached max iterations without a final answer.")

run_tool_agent(
    "Search for the latest GPT-5 news, convert 500 USD to EUR, and calculate 500 * 0.92."
)
```

## Notes
Parallel tool calling is one of the highest-leverage optimizations in agentic systems. If the model requests 3 independent tool calls, executing them sequentially adds 2x-3x unnecessary latency — always parallelize with `asyncio.gather` or a thread pool. One real-world gotcha: if your tool implementations have side effects (database writes, emails), a parallel execution model requires that you think carefully about partial failures — if one tool fails, should the others roll back? For idempotent read-only tools (search, calculations), parallel execution is always safe. Also, Anthropic Claude requires tools to be passed even on follow-up turns in a conversation; forgetting to include the `tools` array on subsequent requests will cause the model to lose awareness of its tools.

## Summary
You can now design and orchestrate multi-tool systems with parallel execution, proper iteration guards, and clean provider-agnostic patterns — the core architecture of production AI agents.
