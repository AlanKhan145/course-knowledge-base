# Repetition Penalty and Stop Sequences

## Overview
Repetition penalty and stop sequences are two complementary controls that prevent common failure modes in LLM output: infinite loops or redundant text (addressed by repetition penalty), and uncontrolled over-generation past a logical endpoint (addressed by stop sequences). Both are critical for building reliable production pipelines where output shape and length must be predictable.

## Key Points
- Repetition penalty (also called frequency penalty or presence penalty depending on the API) reduces the probability of tokens that have already appeared in the context, discouraging the model from repeating itself
- Presence penalty applies a flat one-time reduction to any token that has appeared at least once; frequency penalty scales the reduction proportionally to how many times the token has appeared — they target different repetition patterns
- Stop sequences are one or more exact string patterns that, when generated by the model, immediately terminate the response; the stop string itself is typically not included in the returned text
- Stop sequences are the most reliable way to enforce a strict output boundary — use them to extract structured fields, implement turn-based dialogue, or cap list length
- Setting repetition penalty too high causes the model to avoid important domain-specific words that naturally recur, producing stilted or incoherent output; values between 1.05 and 1.3 are typical safe ranges for most tasks

## Code Example
```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])


# ── Stop Sequences ────────────────────────────────────────────────────────────

def extract_answer_only(question: str) -> str:
    """
    Use a stop sequence to capture only the answer line from a structured
    prompt, stopping before any explanation.
    """
    prompt = (
        f"Answer the following question with a single word or short phrase.\n"
        f"Question: {question}\n"
        f"Answer: "
    )
    response = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=20,
        # Stop as soon as a newline appears — captures only the answer token(s)
        stop_sequences=["\n", "."],
        messages=[{"role": "user", "content": prompt}],
    )
    return response.content[0].text.strip()


def generate_numbered_list(topic: str, max_items: int = 3) -> str:
    """
    Generate exactly N list items by stopping at the N+1 list marker.
    """
    stop_marker = f"{max_items + 1}."   # e.g. "4." when we want 3 items
    prompt = f"List {max_items} facts about {topic}, numbered starting from 1."
    response = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=200,
        stop_sequences=[stop_marker],
        messages=[{"role": "user", "content": prompt}],
    )
    return response.content[0].text.strip()


# ── Repetition Penalty (via OpenAI-compatible endpoint for illustration) ──────
# Anthropic's native API does not expose repetition_penalty directly;
# the example below targets the OpenAI client for APIs that support it.

def demonstrate_repetition_penalty():
    """
    Illustrate the concept using the openai library against an
    OpenAI-compatible endpoint (e.g., local vLLM or Together AI).
    """
    try:
        from openai import OpenAI
        oa_client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY", ""))

        prompt = "Write a short paragraph about the ocean."

        for penalty in [0.0, 1.3]:
            response = oa_client.chat.completions.create(
                model="gpt-4o-mini",
                max_tokens=80,
                frequency_penalty=penalty,  # 0.0 = no penalty, 2.0 = max
                messages=[{"role": "user", "content": prompt}],
            )
            print(f"\nfrequency_penalty={penalty}:")
            print(response.choices[0].message.content.strip())
    except Exception as exc:
        print(f"Skipped OpenAI demo ({exc}). Set OPENAI_API_KEY to enable.")


if __name__ == "__main__":
    print("=== Stop Sequence: Single Answer ===")
    answer = extract_answer_only("What is the capital of Japan?")
    print(f"Extracted answer: '{answer}'")

    print("\n=== Stop Sequence: Capped List ===")
    facts = generate_numbered_list("the Moon", max_items=3)
    print(facts)

    print("\n=== Repetition Penalty Demo ===")
    demonstrate_repetition_penalty()
```

## Notes
Stop sequences are brittle if the model's output format varies — for example, relying on `"\n\n"` as a stop will fail silently if the model uses single newlines. Always pair stop sequences with a reasonable `max_tokens` budget as a fallback safety net. For repetition penalty, note that the Anthropic API does not expose this parameter natively (as of 2024/2025) because Claude's RLHF training substantially reduces repetitive output by default; explicit repetition penalties are more commonly needed with base models or models fine-tuned without RLHF. When working with Hugging Face Transformers or vLLM, use `repetition_penalty` in the generation config rather than trying to implement it manually via logit processors.

## Summary
Repetition penalty reduces looping and redundancy in generated text, while stop sequences give you hard boundaries over where the model's output ends. After this lesson you can use both controls to make LLM outputs more predictable and structured in real applications.
