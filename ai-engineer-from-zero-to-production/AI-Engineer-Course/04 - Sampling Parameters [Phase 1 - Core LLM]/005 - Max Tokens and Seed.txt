# Max Tokens and Seed

## Overview
Max tokens and seed are two foundational infrastructure-level parameters. Max tokens sets a hard ceiling on output length, protecting both latency and cost. Seed enables reproducible generation, which is indispensable for debugging, evaluation, and regression testing. Together they give AI Engineers control over the operational boundaries of model inference.

## Key Points
- `max_tokens` (also called `max_new_tokens` in Hugging Face) specifies the maximum number of tokens the model may generate in a single call; the model will stop at this limit even if the output is mid-sentence
- Tokens are not words: on average, one token is roughly 0.75 English words; "ChatGPT" is 1 token, "antidisestablishmentarianism" may be 5-6 tokens depending on the tokenizer
- Setting `max_tokens` too low is a common production bug — it truncates responses silently and the calling code must explicitly check the `stop_reason` field (`"max_tokens"` vs. `"end_turn"`) to detect truncation
- `seed` fixes the random number generator state on the server side; given the same prompt, temperature, and seed, the model should produce identical (or near-identical) output across calls — this is called deterministic inference
- Seed reproducibility is not perfectly guaranteed across different API versions, hardware, or batching configurations, but it is reliable enough for testing and evaluation workflows within a stable deployment

## Code Example
```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

PROMPT = "Explain what a neural network is in exactly two sentences."


# ── Max Tokens ────────────────────────────────────────────────────────────────

def call_with_max_tokens(max_tokens: int) -> dict:
    """Return the response text and stop reason for a given max_tokens value."""
    response = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=max_tokens,
        messages=[{"role": "user", "content": PROMPT}],
    )
    return {
        "max_tokens": max_tokens,
        "stop_reason": response.stop_reason,   # "end_turn" or "max_tokens"
        "output_tokens": response.usage.output_tokens,
        "text": response.content[0].text,
    }


def demonstrate_max_tokens():
    print("=== Max Tokens Comparison ===\n")
    for limit in [10, 50, 300]:
        result = call_with_max_tokens(limit)
        print(f"max_tokens={result['max_tokens']} | "
              f"stop_reason={result['stop_reason']} | "
              f"tokens_used={result['output_tokens']}")
        print(f"  Text: {result['text'][:120]}{'...' if len(result['text']) > 120 else ''}")
        if result["stop_reason"] == "max_tokens":
            print("  WARNING: Response was truncated — increase max_tokens or shorten prompt.")
        print()


# ── Seed ─────────────────────────────────────────────────────────────────────

def call_with_seed(seed: int | None, temperature: float = 0.9) -> str:
    """Generate a response, optionally fixing the seed for reproducibility."""
    # Note: seed support varies by provider. The OpenAI client is used here
    # because it exposes seed directly; Anthropic's Python SDK supports it
    # via the 'seed' parameter in recent versions.
    response = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=60,
        temperature=temperature,
        # seed=seed,   # Uncomment when seed is supported in your SDK version
        messages=[{"role": "user", "content": "Name one random planet in our solar system."}],
    )
    return response.content[0].text.strip()


def demonstrate_seed():
    """
    Show reproducibility by calling twice with the same parameters.
    Since Anthropic's seed support depends on SDK version, this demo
    calls without seed to show natural variance, then explains the concept.
    """
    print("=== Seed / Reproducibility Demo ===\n")
    print("Without seed (natural variance across calls):")
    for i in range(3):
        result = call_with_seed(seed=None, temperature=1.0)
        print(f"  Call {i+1}: {result}")

    print("\nWith seed=42 (should produce consistent output across calls):")
    # Illustrate using OpenAI client which has broad seed support
    try:
        from openai import OpenAI
        oa = OpenAI(api_key=os.environ.get("OPENAI_API_KEY", ""))
        for i in range(3):
            r = oa.chat.completions.create(
                model="gpt-4o-mini",
                max_tokens=20,
                temperature=1.0,
                seed=42,
                messages=[{"role": "user",
                           "content": "Name one random planet in our solar system."}],
            )
            print(f"  Call {i+1}: {r.choices[0].message.content.strip()}")
    except Exception as exc:
        print(f"  (Set OPENAI_API_KEY to run seed demo: {exc})")


# ── Token estimation utility ──────────────────────────────────────────────────

def estimate_token_budget(text: str, words_per_token: float = 0.75) -> int:
    """Rough heuristic: words / 0.75 ≈ tokens. Use tiktoken for precision."""
    word_count = len(text.split())
    return int(word_count / words_per_token)


if __name__ == "__main__":
    demonstrate_max_tokens()
    demonstrate_seed()

    sample_text = "The quick brown fox jumps over the lazy dog."
    estimate = estimate_token_budget(sample_text)
    print(f"\nEstimated tokens for sample text: ~{estimate} "
          f"(actual may differ by tokenizer)")
```

## Notes
A frequently overlooked production issue: APIs bill for output tokens actually generated, not for `max_tokens`. Setting `max_tokens` too high does not cost extra on its own, but it does increase worst-case latency because the server reserves a KV-cache slot for the full budget. For real-time streaming applications, set `max_tokens` to a realistic ceiling based on your p99 expected output length rather than a large round number. Regarding seed: even with identical seeds, output can differ across model version upgrades — so never rely on seeds for long-term reproducibility (e.g., content that must remain byte-identical over months). Use seeds for short-lived test fixtures, evaluation snapshots, and debugging sessions only.

## Summary
`max_tokens` prevents runaway generation and controls cost; `seed` enables reproducible outputs for testing and debugging. After this lesson you can configure both parameters deliberately and detect truncated responses programmatically using the `stop_reason` field.
