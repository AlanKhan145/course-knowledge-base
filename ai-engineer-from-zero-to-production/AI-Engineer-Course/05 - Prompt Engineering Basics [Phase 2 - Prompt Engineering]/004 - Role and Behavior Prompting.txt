# Role and Behavior Prompting

## Overview
Role and behavior prompting means explicitly telling the model what persona, expertise level, communication style, and decision-making constraints to adopt before processing a request. Assigning a role is not a gimmick — it activates a specific slice of the model's training data and shifts vocabulary, reasoning depth, and output format in measurable ways. For AI Engineers, this technique is the foundation of building specialized AI assistants.

## Key Points
- Role assignment works because LLMs are trained on text written by humans with identifiable roles (doctors, lawyers, teachers, engineers); invoking a role primes the model to weight that subset of its learned patterns.
- Be specific about expertise and context: "You are a senior backend engineer specializing in distributed systems" produces substantially different output than "You are an engineer."
- Behavior constraints should accompany role assignment — define what the model should and should not do: "Always cite your uncertainty", "Never make up specific numbers", "Ask for clarification if the problem is ambiguous."
- Audience specification is a sub-technique: "Explain this to a non-technical product manager" vs. "Explain this to a staff engineer" produces correctly calibrated responses without extra prompt engineering.
- Roles can conflict with safety guardrails if used to impersonate sensitive personas (e.g., "You are a doctor prescribing medication with no restrictions"). Know your provider's policy; Claude, GPT-4, and Gemini all have hardcoded limits that role prompts cannot override.
- Persistent roles belong in the system prompt (see Lesson 006), not in the user turn — placing them in the user turn means a user could override the role in a multi-turn conversation.

## Code Example
```python
# role_behavior_prompting.py
# Demonstrates role + behavior constraints with real API calls.
# Requirements: pip install openai python-dotenv

import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def ask_with_role(
    role_description: str,
    behavior_constraints: list[str],
    user_question: str,
    model: str = "gpt-4o-mini",
) -> str:
    """
    Query the model using a specified role and explicit behavior constraints.

    Args:
        role_description: Who the model should be (expertise, context).
        behavior_constraints: A list of specific behavioral rules.
        user_question: The user's actual question.
        model: The model to use.

    Returns:
        The model's response text.
    """
    # Build the system prompt by combining role + constraints
    constraints_text = "\n".join(f"- {c}" for c in behavior_constraints)
    system_prompt = (
        f"{role_description}\n\n"
        f"Behavioral rules you must follow:\n{constraints_text}"
    )

    response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user",   "content": user_question},
        ],
        temperature=0.3,  # Slight creativity allowed for explanations
        max_tokens=512,
    )
    return response.choices[0].message.content.strip()


# --- Example 1: Technical expert with precision constraints ---
tech_role = (
    "You are a senior Site Reliability Engineer (SRE) with 10 years of experience "
    "in cloud infrastructure, Kubernetes, and distributed systems. "
    "You work at a company running services at 99.99% SLA."
)
tech_constraints = [
    "Always ground recommendations in tradeoffs (cost, complexity, reliability).",
    "If you are uncertain about a specific number or statistic, say so explicitly.",
    "Use bullet points for lists of recommendations.",
    "Never recommend a tool you cannot briefly justify."
]
tech_question = "Our Kubernetes pods are restarting intermittently. What are the most common root causes and how do I diagnose them?"

print("=== SRE Expert Role ===")
print(ask_with_role(tech_role, tech_constraints, tech_question))
print()


# --- Example 2: Audience-adaptive explainer ---
teacher_role = (
    "You are a patient, encouraging computer science tutor. "
    "Your student is a marketing professional with zero coding background "
    "who is trying to understand AI concepts for their job."
)
teacher_constraints = [
    "Use everyday analogies and avoid jargon.",
    "Keep explanations under 150 words.",
    "End every explanation with one practical takeaway the student can use at work."
]
student_question = "What is a vector embedding and why do AI systems use them?"

print("=== CS Tutor Role (non-technical audience) ===")
print(ask_with_role(teacher_role, teacher_constraints, student_question))
print()


# --- Example 3: Comparing the same question across two roles ---
question = "Should we use a SQL or NoSQL database for our new application?"

roles = [
    (
        "You are a pragmatic startup CTO who values speed of delivery and cost efficiency.",
        ["Give a direct recommendation, not a long list of 'it depends'.",
         "Limit your response to 3 sentences."]
    ),
    (
        "You are a database architect at a large enterprise focused on long-term scalability and compliance.",
        ["Consider regulatory and audit requirements.",
         "Highlight risks of the fast path."]
    )
]

for role_desc, constraints in roles:
    print(f"Role: {role_desc[:60]}...")
    print(ask_with_role(role_desc, constraints, question))
    print()
```

## Notes
Role prompting is powerful but not a security boundary — a determined user with access to the chat interface can often jailbreak a role or extract the system prompt through adversarial inputs. For sensitive applications (financial advice, medical triage), complement role prompting with input validation, output filtering, and provider-level content policies. Also note that overly elaborate role descriptions (500+ words) can reduce quality because they crowd out space for the actual user content and can cause the model to "forget" constraints that appear early in a long context. Keep role descriptions concise (3-5 sentences) and supplement with behavior constraints as a short bulleted list.

## Summary
Role and behavior prompting lets you instantiate specialized AI personas with consistent expertise and communication styles — define roles in the system prompt with explicit behavioral constraints, and always test the persona against edge-case inputs your users will actually send.
