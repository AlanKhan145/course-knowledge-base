# What Is an AI Engineer

## Overview
This lesson defines the AI Engineer role as it exists in the industry in 2024/2025 — distinct from both software engineers and ML researchers. You will learn what AI Engineers actually build, what skills they must master, and why this role has become one of the fastest-growing positions in tech.

## Key Points
- An AI Engineer is a software engineer who specializes in integrating, orchestrating, and deploying AI models (primarily large language models) into production applications — they consume AI, they do not train it from scratch.
- The core skill set is: API integration, prompt design, retrieval systems (RAG), agent architecture, evaluation, and MLOps-style observability — not gradient descent or CUDA kernels.
- AI Engineers work at the application layer: they wire together LLMs, vector databases, external tools, and business logic into coherent systems using frameworks like LangChain, LlamaIndex, or custom orchestration code.
- Reliability engineering is central to the role — LLM outputs are non-deterministic, so AI Engineers must build evaluation pipelines, fallback logic, retry strategies, and monitoring dashboards.
- The role emerged around 2022/2023 with the mainstream availability of GPT-3.5/4 APIs; by 2024 it is a recognized job title at companies ranging from startups to Fortune 500 enterprises.

## Code Example
```python
# Minimal working demo: what an AI Engineer actually writes day-to-day
# This is a production-style wrapper around an LLM API call

import os
from openai import OpenAI
from dotenv import load_dotenv

# Load API key from .env file — never hardcode credentials
load_dotenv()

client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def ask_llm(user_question: str, system_prompt: str = "You are a helpful assistant.") -> str:
    """
    A minimal but production-aware LLM call:
    - Separates system and user roles
    - Returns just the text content
    - Raises clearly on API errors
    """
    response = client.chat.completions.create(
        model="gpt-4o-mini",          # Cost-efficient model for most tasks
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_question},
        ],
        temperature=0.2,               # Low temperature = more deterministic output
        max_tokens=512,                # Always set a ceiling to control cost
    )
    return response.choices[0].message.content


if __name__ == "__main__":
    answer = ask_llm("What does an AI Engineer do in 2024?")
    print(answer)
```

## Notes
The term "AI Engineer" is sometimes used interchangeably with "LLM Engineer" or "Generative AI Engineer" — all refer to roughly the same application-layer role. Do not confuse this with "AI/ML Engineer" which often implies training and fine-tuning models, requiring much deeper math. In practice, 80% of business value from AI comes from good prompt engineering, retrieval, and orchestration — not from custom model training. The most important non-technical skill for this role is the ability to decompose a business problem into an AI-solvable sub-problem.

## Summary
An AI Engineer builds production systems on top of AI models — their primary tools are APIs, orchestration frameworks, vector databases, and evaluation pipelines, and their primary responsibility is making AI reliable enough to ship to real users.
