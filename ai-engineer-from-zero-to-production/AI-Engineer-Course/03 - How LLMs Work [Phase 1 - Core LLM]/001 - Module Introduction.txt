# Module Introduction — How LLMs Work

## Overview
This module provides the essential mental model every AI Engineer needs before writing a single line of production code. You will learn how Large Language Models actually function under the hood — from tokenization and context windows to hallucination and generation parameters — so that every design decision you make later is grounded in first principles rather than guesswork.

## Key Points
- This module covers the internal mechanics of LLMs that directly affect how you architect, prompt, and debug AI applications in production.
- Understanding tokenization prevents costly surprises in billing, context limits, and truncation bugs.
- Knowing how the prompt-to-completion pipeline works lets you reason about latency, streaming, and failure modes.
- The distinction between inference, training, and fine-tuning clarifies when you need a different model versus a better prompt.
- Hallucination is not a bug to be patched — it is a structural property of how LLMs generate text, and engineers must design around it.
- Generation parameters (temperature, top-p, etc.) give you direct control over output quality and reproducibility.

## What This Module Covers
- [ ] What Is a Large Language Model — probability distributions over tokens, not a database of facts
- [ ] Tokenization — how raw text becomes numbers, why token count matters for cost and limits
- [ ] Context Window — what the model can "see," why it matters, and current practical limits
- [ ] The Prompt to Completion Pipeline — step-by-step journey from API call to generated text
- [ ] Inference, Training, and Fine-tuning — three distinct phases and when each applies
- [ ] Hallucination in LLMs — root causes and mitigation strategies for production systems
- [ ] Deterministic vs Creative Generation — controlling randomness with temperature and sampling
- [ ] Lab: Token Cost Estimator — a practical tool you will use on every project going forward

## Summary
By the end of this module you will have a working mental model of how LLMs process and generate text, which is the prerequisite for every advanced topic in this course — RAG, agents, evaluation, and production deployment.
