## 1. Tom tat

Bai nay giai thich **Bug Report** trong boi canh QA Engineer Roadmap 2026. Sau bai hoc, ban nen biet topic nay nam o dau trong quy trinh QA, manual testing, automation, API, non-functional testing, reporting, monitoring hoac CI/CD.

## 2. Muc tieu hoc tap

- Giai thich duoc Bug Report bang ngon ngu cua ban.
- Nhan biet topic nay anh huong den risk, coverage, defect detection, release confidence hoac production quality nao.
- Ap dung vao mot artifact nho: test case, checklist, bug report, API collection, automation test, report, dashboard note hoac README.

## 3. Khai niem chinh

- A bug report should make the defect easy to reproduce, understand and prioritize.
- It should include environment, steps, expected result, actual result, evidence, severity and priority.
- Strong bug reports reduce back-and-forth and speed up fixes.

## 4. Thuc hanh

1. Create a checklist item for the topic.
2. Apply it to one screen, request, log, report or device scenario.
3. Capture evidence: screenshot, log line, report, table or note.

## 5. Bai tap

Explain the concept, create a small QA artifact and write how it affects release confidence.

## 6. Checklist hoan thanh

- [ ] Co dinh nghia ngan gon.
- [ ] Co vi du trong QA/testing workflow.
- [ ] Co artifact nho de dua vao portfolio.
- [ ] Co ghi chu ve risk, evidence, automation stability, CI/CD hoac release decision neu lien quan.

## 7. Ghi chu release

Khi dung trong release thuc te, hay hoi: bug nao co the lot qua, evidence nao chung minh da test, test co tai lap duoc khong, ai can xem report va ket qua nay co thay doi quyet dinh release khong.
