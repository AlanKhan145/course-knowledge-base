## 1. Tom tat

Bai nay giai thich **Edge Case Thinking** trong boi canh QA Engineer Roadmap 2026. Sau bai hoc, ban nen biet topic nay nam o dau trong quy trinh QA, manual testing, automation, API, non-functional testing, reporting, monitoring hoac CI/CD.

## 2. Muc tieu hoc tap

- Giai thich duoc Edge Case Thinking bang ngon ngu cua ban.
- Nhan biet topic nay anh huong den risk, coverage, defect detection, release confidence hoac production quality nao.
- Ap dung vao mot artifact nho: test case, checklist, bug report, API collection, automation test, report, dashboard note hoac README.

## 3. Khai niem chinh

- Edge Case Thinking is part of the QA Mindset topic in the QA Engineer roadmap.
- Practice by turning one happy path into multiple edge cases.
- Write risks in user language before writing test steps.
- Learn it by connecting the concept to risk, evidence, user impact and release confidence.
- A QA Engineer should know how to turn this topic into a test artifact or investigation note.

## 4. Thuc hanh

1. Write a 5-line note explaining Edge Case Thinking.
2. Add one QA example or edge case.
3. Create a small artifact: checklist, test case, bug report, command or README note.

## 5. Bai tap

Explain the concept, create a small QA artifact and write how it affects release confidence.

## 6. Checklist hoan thanh

- [ ] Co dinh nghia ngan gon.
- [ ] Co vi du trong QA/testing workflow.
- [ ] Co artifact nho de dua vao portfolio.
- [ ] Co ghi chu ve risk, evidence, automation stability, CI/CD hoac release decision neu lien quan.

## 7. Ghi chu release

Khi dung trong release thuc te, hay hoi: bug nao co the lot qua, evidence nao chung minh da test, test co tai lap duoc khong, ai can xem report va ket qua nay co thay doi quyet dinh release khong.
