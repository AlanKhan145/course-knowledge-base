## 1. Tom tat

Bai nay giai thich **OWASP Top 10** trong boi canh QA Engineer Roadmap 2026. Sau bai hoc, ban nen biet topic nay nam o dau trong quy trinh QA, manual testing, automation, API, non-functional testing, reporting, monitoring hoac CI/CD.

## 2. Muc tieu hoc tap

- Giai thich duoc OWASP Top 10 bang ngon ngu cua ban.
- Nhan biet topic nay anh huong den risk, coverage, defect detection, release confidence hoac production quality nao.
- Ap dung vao mot artifact nho: test case, checklist, bug report, API collection, automation test, report, dashboard note hoac README.

## 3. Khai niem chinh

- OWASP Top 10 summarizes common web application security risks.
- QA Engineers do not need to be pentesters to test access control, input validation and authentication basics.
- Security test cases should include both expected access and forbidden access.

## 4. Thuc hanh

1. Write one happy path and one forbidden-path test.
2. Capture expected status/message or UI behavior.
3. Document impact if the check fails.

## 5. Bai tap

Write a security test case that verifies a forbidden action is blocked.

## 6. Checklist hoan thanh

- [ ] Co dinh nghia ngan gon.

- [ ] Co artifact nho de dua vao portfolio.
- [ ] Co ghi chu ve risk, evidence, automation stability, CI/CD hoac release decision neu lien quan.

## 7. Ghi chu release

Khi dung trong release thuc te, hay hoi: bug nao co the lot qua, evidence nao chung minh da test, test co tai lap duoc khong, ai can xem report va ket qua nay co thay doi quyet dinh release khong.
