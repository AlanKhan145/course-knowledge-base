# 019 - Module Introduction

## Section

Key Features & Efficient Usage

## Main Idea

This lecture introduces Section 2, which is the core of the course. Students learn what topics will be covered in this section and why these features are what separate basic Claude Code usage from genuinely productive AI-assisted development.

## Learning Objectives

By the end of this lecture, students should understand:

* What topics Section 2 covers.
* Why these features matter for real development workflows.
* How the features in this section connect to each other.
* What the overall learning goal of this section is.

## Key Concepts

### Section 2 Topics Overview

Section 2 covers the main productivity features of Claude Code:

1. **Prompt and context engineering** — Getting better results by providing better input.
2. **Specs** — Structured feature descriptions for complex tasks.
3. **`CLAUDE.md`** — Project memory and instruction files.
4. **Plan Mode** — Structured implementation planning before code edits.
5. **Built-in tools** — File reading, searching, editing, command execution.
6. **MCP servers** — Connecting Claude to external tools and data sources.
7. **Subagents** — Specialized assistants for specific types of tasks.
8. **Agent skills** — Reusable workflows and instruction sets.
9. **Custom commands** — Prompt templates for repeated tasks.
10. **Hooks** — Automated actions triggered during Claude workflows.
11. **Plugins** — Extensions that add new capabilities.
12. **Browser feedback** — Using the browser as a feedback mechanism for UI work.
13. **Automated tests** — Using test results as feedback to guide Claude.
14. **Agent loops** — Running Claude in iterative improvement cycles.

### Why These Features Matter

Basic Claude Code usage (just asking questions and getting code) leaves most of the value untapped. The features in this section unlock:

* More consistent results through better context.
* Safer implementation through planning before editing.
* More automation through subagents, skills, and hooks.
* Better feedback through tests and browser access.
* Reusability through commands and skills.

### How Features Connect

The features in this section build on each other:

* Context engineering makes prompts better.
* `CLAUDE.md` provides persistent context.
* Plan Mode makes implementation safer.
* Subagents and skills make workflows reusable.
* Hooks and feedback loops make Claude more autonomous and self-correcting.

## Important Notes

* Section 2 is the longest and most important section of the course.
* Students should complete Section 1 before starting here.
* Not all features are needed for every project. Students should identify which ones apply to their use cases.
* Some features (MCP servers, hooks) require additional setup and configuration.

## Why This Lecture Matters

A brief introduction to the section helps students know what is coming, understand the relationships between features, and mentally prepare for more complex material.

## Summary

Section 2 covers the core productivity features of Claude Code: context engineering, specs, `CLAUDE.md`, Plan Mode, tools, MCP servers, subagents, skills, commands, hooks, plugins, browser feedback, automated tests, and agent loops.
