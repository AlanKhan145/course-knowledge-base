# 020 - Internet Protocol

**Section:** 03 - Protocols  
**Duration:** 1hr 7min  

---

## 1. Summary

The **Internet Protocol**, usually called **IP**, is one of the most important protocols in networking.

IP is the protocol responsible for moving data from one machine to another using **IP addresses**.

No matter what application protocol you use, the data will eventually be placed inside an **IP packet**.

Examples of application or transport protocols that eventually use IP:

- HTTP
- HTTPS
- gRPC
- TCP
- UDP
- DNS
- PostgreSQL traffic
- MySQL traffic
- SSH

At the IP layer, we do not care about ports, HTTP headers, JSON, encryption, or application logic.

At this layer, the main concern is simple:

```text
Where is this packet coming from?
Where is this packet going?
```

The Internet Protocol lives at **Layer 3**, the **Network Layer**, in the OSI Model.

---

## 2. Key Concepts

## 2.1 IP Is the Vehicle of the Internet

IP can be thought of as the vehicle that carries data across networks.

Application data is wrapped inside transport data, and transport data is eventually wrapped inside an IP packet.

Example:

```text
HTTP request
→ TCP segment
→ IP packet
→ Ethernet/Wi-Fi frame
→ Physical signal
```

Routers mainly care about the IP packet.

They usually do not care whether the packet contains:

- HTTP
- gRPC
- TCP
- UDP
- Database traffic
- Encrypted data

To a router, it is mostly just an IP packet with:

```text
Source IP address
Destination IP address
Data
```

---

## 2.2 IP Address

An IP address is a **Layer 3 property**.

It identifies a machine or network interface on a network.

An IPv4 address has **4 bytes**, usually written like this:

```text
192.168.1.23
```

Each part is one byte:

```text
192     168     1     23
byte 1  byte 2  byte 3 byte 4
```

An IP address can be assigned in two ways:

- Automatically
- Statically

Automatic assignment is usually done using **DHCP**.

Static assignment means the IP address is manually configured.

---

## 2.3 Network Portion and Host Portion

An IP address has two parts:

```text
Network portion
Host portion
```

Example:

```text
192.168.254.0/24
```

The `/24` means the first 24 bits are the network portion.

Since each byte has 8 bits:

```text
8 + 8 + 8 = 24
```

So in this example:

```text
192.168.254 = network portion
0           = host portion
```

This means all addresses that start with `192.168.254` belong to the same subnet.

Example hosts in the same subnet:

```text
192.168.254.1
192.168.254.2
192.168.254.100
192.168.254.200
```

---

## 2.4 Subnet

A **subnet** is a smaller logical network inside a larger network.

Example:

```text
192.168.1.0/24
```

This subnet can contain hosts such as:

```text
192.168.1.1
192.168.1.2
192.168.1.3
...
192.168.1.254
```

The subnet helps a machine decide whether the destination is:

```text
Inside the same local network
```

or

```text
Outside the local network
```

---

## 2.5 Subnet Mask

A subnet mask is used to determine whether two IP addresses are in the same subnet.

Example subnet mask:

```text
255.255.255.0
```

This corresponds to:

```text
/24
```

A machine uses the subnet mask to compare:

```text
Its own IP address
Destination IP address
```

If both addresses produce the same network address after applying the subnet mask, they are in the same subnet.

---

## 2.6 Same Subnet Example

Host A wants to talk to Host B.

```text
Host A: 192.168.1.23
Host B: 192.168.1.2
Subnet mask: 255.255.255.0
```

Apply the subnet mask:

```text
192.168.1.23 AND 255.255.255.0 = 192.168.1.0
192.168.1.2  AND 255.255.255.0 = 192.168.1.0
```

Both results are the same:

```text
192.168.1.0
```

Therefore, both hosts are in the same subnet.

Result:

```text
No router is needed.
The packet can be sent directly inside the local network.
```

The machine still needs the destination MAC address, so it may use ARP to find it.

---

## 2.7 Different Subnet Example

Host A wants to talk to Host B.

```text
Host A: 192.168.1.23
Host B: 192.168.2.2
Subnet mask: 255.255.255.0
```

Apply the subnet mask:

```text
192.168.1.23 AND 255.255.255.0 = 192.168.1.0
192.168.2.2  AND 255.255.255.0 = 192.168.2.0
```

The results are different:

```text
192.168.1.0
192.168.2.0
```

Therefore, the destination is not in the same subnet.

Result:

```text
The packet must be sent to the default gateway.
```

---

## 2.8 Default Gateway

A **default gateway** is the device that receives packets when the destination is outside the current subnet.

Most of the time, the default gateway is a router.

A host usually needs three important network settings:

```text
IP address
Subnet mask
Default gateway
```

Example:

```text
IP address:      192.168.1.23
Subnet mask:     255.255.255.0
Default gateway: 192.168.1.100
```

If the destination is not in the same subnet, the host sends the packet to:

```text
192.168.1.100
```

The router then decides where to forward the packet next.

---

## 2.9 Router

A router connects different networks together.

A router usually has multiple network interfaces.

Example:

```text
Router interface 1: 192.168.1.100
Router interface 2: 192.168.2.200
```

This router can connect:

```text
192.168.1.0/24
```

with:

```text
192.168.2.0/24
```

A router lives in multiple networks at the same time.

That is why it can forward packets between different subnets.

---

## 2.10 ARP and MAC Address

When a host wants to send data to another machine in the same local network, it needs the destination MAC address.

IP address is Layer 3.

MAC address is Layer 2.

To find the MAC address of a known IP address, the host may use **ARP**.

Example:

```text
Who has 192.168.1.100?
Tell 192.168.1.23.
```

The machine with that IP replies with its MAC address.

This is important when sending packets to:

- Another host in the same subnet
- The default gateway

---

## 2.11 ARP Poisoning

ARP poisoning is an attack where a malicious machine pretends to be another machine, often the default gateway.

Example:

```text
Attacker: I am the router.
```

If other machines believe this false ARP information, their traffic may be sent through the attacker.

This can allow the attacker to inspect or modify network traffic.

---

## 3. IP Packet Anatomy

## 3.1 IP Packet Structure

An IP packet has two main parts:

```text
Header
Data
```

The header contains metadata.

The data section contains the actual payload.

Example:

```text
+-------------------+-------------------+
| IP Header          | Data              |
+-------------------+-------------------+
```

The data can contain many different things:

- TCP segment
- UDP datagram
- ICMP message
- Encrypted data
- Application data indirectly

---

## 3.2 IP Header Size

The minimum IPv4 header size is:

```text
20 bytes
```

The maximum IPv4 header size is:

```text
60 bytes
```

The header becomes larger if IP options are used.

This header is overhead.

For example, sending 1 byte of data with a 20-byte IP header is inefficient.

That is why networking systems often try to avoid sending tiny packets too frequently.

---

## 3.3 Maximum IP Packet Size

The total length field in an IPv4 packet is 16 bits.

That means the theoretical maximum IP packet size is:

```text
65,535 bytes
```

However, in real networks, packets are usually much smaller because of the **MTU**.

---

## 3.4 MTU

MTU stands for:

```text
Maximum Transmission Unit
```

It usually refers to the maximum size of a frame that can be transmitted on a network link.

A common MTU size on Ethernet is:

```text
1500 bytes
```

If an IP packet is larger than the MTU, it may need to be fragmented.

---

## 3.5 Fragmentation

Fragmentation happens when one IP packet is too large to fit into one frame.

Example:

```text
IP packet size: 2000 bytes
MTU:            1500 bytes
```

The packet cannot fit into one frame.

There are two options:

```text
Option 1: Fragment the packet
Option 2: Drop the packet and send an ICMP error
```

Fragmentation splits one IP packet into multiple smaller fragments.

Example:

```text
Original IP packet
→ Fragment 1
→ Fragment 2
```

The receiver must reassemble the fragments.

Fragmentation can cause problems because:

- Fragments may arrive out of order
- Some fragments may be lost
- Reassembly costs resources
- Attackers may abuse fragmentation
- Some protocols avoid fragmentation completely

Modern systems often prefer avoiding fragmentation.

---

## 3.6 Don't Fragment Flag

The IP header can include a flag that means:

```text
Do not fragment this packet.
```

If this flag is set and a router cannot forward the packet because it is too large for the MTU, the router should drop the packet and send an ICMP message back.

This message means:

```text
Fragmentation needed.
```

This is important for discovering the correct packet size along a network path.

---

## 3.7 Time To Live

TTL stands for:

```text
Time To Live
```

TTL prevents packets from traveling around the Internet forever.

Every IP packet has a TTL value.

Example:

```text
TTL = 64
```

Each router that forwards the packet decreases the TTL by 1.

Example:

```text
64 → 63 → 62 → 61
```

If TTL reaches zero, the router drops the packet.

Then it usually sends an ICMP message back to the source.

TTL solves the problem of routing loops.

Without TTL, a packet could loop forever between routers.

---

## 3.8 Protocol Field

The IP header has a protocol field.

This field tells what kind of data is inside the IP packet.

Examples:

```text
TCP
UDP
ICMP
```

This helps routers and hosts understand what the packet contains without reading the entire payload.

It is metadata for performance and routing decisions.

---

## 3.9 Source IP and Destination IP

The most important fields in the IP header are:

```text
Source IP address
Destination IP address
```

Example:

```text
Source IP:      192.168.1.23
Destination IP: 8.8.8.8
```

The destination IP tells routers where the packet should go.

The source IP tells the receiver where the packet came from.

---

## 3.10 IP Spoofing

IP spoofing means changing the source IP address of a packet.

In theory, a machine can create a packet with a fake source IP.

However, in practice, most ISPs and network providers block this behavior.

Why?

Because the ISP knows which IP address belongs to the customer.

If a packet comes from the wrong source IP, the ISP can reject it.

Also, if the source IP is fake, responses will not return to the attacker correctly.

---

## 3.11 Explicit Congestion Notification

Explicit Congestion Notification, or **ECN**, is a mechanism used to signal network congestion without dropping packets.

Congestion happens when routers receive more packets than they can process.

Routers have buffers.

If the buffer becomes full, packets may be dropped.

Traditional congestion detection works like this:

```text
Packet is dropped
→ Sender notices timeout
→ Sender assumes congestion
```

ECN improves this by allowing a router to mark a packet instead of dropping it.

The receiver can then inform the sender that congestion was experienced.

This allows the sender to slow down before packet loss happens.

---

## 4. ICMP

## 4.1 What Is ICMP?

ICMP stands for:

```text
Internet Control Message Protocol
```

ICMP is a Layer 3 protocol.

It is used to send control and informational messages between hosts and routers.

ICMP does not use ports.

Ports belong to Layer 4.

ICMP works directly with IP.

---

## 4.2 What Is ICMP Used For?

ICMP is used for messages such as:

- Host unreachable
- Network unreachable
- Port unreachable
- Fragmentation needed
- TTL expired
- Echo request
- Echo reply

Tools that use ICMP:

- ping
- traceroute

---

## 4.3 ICMP Does Not Need a Listener Port

Unlike TCP or UDP, ICMP does not require an application to listen on a port.

There is no ICMP port.

If a host supports ICMP, it can respond to ICMP messages directly at the network layer.

This is why ping can work without any application server running.

---

## 4.4 Ping

`ping` is a tool used to test whether a host is reachable.

Ping uses ICMP.

It sends:

```text
ICMP Echo Request
```

The destination replies with:

```text
ICMP Echo Reply
```

Example:

```bash
ping google.com
```

Before ping can send packets to `google.com`, it must resolve the domain name to an IP address using DNS.

Then it sends ICMP packets to that IP.

---

## 4.5 Ping Example

A client sends an ICMP echo request.

```text
Client: 192.168.1.23
Server: 10.0.0.3
```

The packet travels through routers.

Each router decreases the TTL.

If the destination supports ICMP, it replies with an ICMP echo reply.

The client can measure the round-trip time.

Example output concept:

```text
64 bytes from 8.8.8.8: icmp_seq=1 ttl=118 time=12 ms
```

Important fields:

```text
icmp_seq = sequence number
ttl      = time to live
time     = round-trip time
```

---

## 4.6 Why Ping May Fail

Ping may fail even if the server is alive.

Reasons include:

- ICMP is blocked by firewall
- ICMP is blocked by router
- Host does not respond to ICMP
- Network path drops ICMP packets
- Security policy disables ping

So this is not always true:

```text
Ping failed = server is down
```

A better interpretation is:

```text
Ping failed = ICMP response was not received
```

---

## 4.7 Traceroute

`traceroute` is a tool used to discover the path that packets take through the network.

On Linux and macOS:

```bash
traceroute google.com
```

On Windows:

```bash
tracert google.com
```

Traceroute uses TTL and ICMP.

---

## 4.8 How Traceroute Works

Traceroute sends packets with small TTL values.

First packet:

```text
TTL = 1
```

The first router decrements TTL to zero and drops the packet.

Then the router sends back an ICMP message.

Now traceroute knows the first hop.

Second packet:

```text
TTL = 2
```

The packet passes the first router, reaches the second router, then TTL becomes zero.

The second router sends back an ICMP message.

Now traceroute knows the second hop.

This continues until the packet reaches the destination.

Example flow:

```text
TTL 1 → first router replies
TTL 2 → second router replies
TTL 3 → third router replies
TTL 4 → fourth router replies
...
```

This is how traceroute builds the path.

---

## 4.9 Why Traceroute May Be Incomplete

Traceroute does not always show the full path.

Reasons:

- ICMP may be blocked
- Some routers do not respond
- Firewalls may drop traceroute packets
- Network paths can change
- Different packets may take different routes

In traceroute output, missing routers may appear as:

```text
* * *
```

This usually means no response was received from that hop.

---

## 4.10 TCP Black Hole Problem

Blocking ICMP can cause hidden networking problems.

Example:

1. TCP handshake succeeds.
2. The connection appears open.
3. The client starts sending larger packets.
4. A router requires fragmentation or a smaller packet size.
5. The router sends an ICMP message saying fragmentation is needed.
6. ICMP is blocked.
7. The client never receives the message.
8. Data stops flowing.

This can create a situation where the connection exists, but data does not move correctly.

This is sometimes called a TCP black hole problem.

---

## 5. Important Notes

## 5.1 IP Is Layer 3

IP belongs to the Network Layer.

At this layer, there are:

```text
IP addresses
Packets
Routing
TTL
ICMP
```

There are no:

```text
Ports
HTTP headers
JSON objects
Application routes
```

Ports belong to Layer 4.

HTTP belongs to Layer 7.

---

## 5.2 Routers Care About IP Packets

Routers mainly inspect the IP header.

They care about:

```text
Destination IP
TTL
Protocol field
Fragmentation flags
```

They usually do not care about application data.

---

## 5.3 Same Subnet vs Different Subnet

If the destination is in the same subnet:

```text
Send directly using Layer 2 communication.
```

If the destination is in a different subnet:

```text
Send to the default gateway.
```

This is one of the most important ideas in IP networking.

---

## 5.4 Database and Backend Network Design

If a backend application and database are in different subnets, traffic may need to pass through a router.

If that router is congested, database requests may experience extra latency.

Better design:

```text
Backend application and database should often be placed close together in the network.
```

For high-performance systems, a fast switch may be better than routing internal traffic unnecessarily.

---

## 5.5 IP Is Stateless

IP itself is stateless.

Routers do not usually remember every packet they have seen.

Each packet carries the information needed to route it.

That is why fields like TTL are important.

TTL carries state inside the packet itself.

---

## 5.6 ICMP Is Important

ICMP is not just for ping.

It is used for important network control messages.

Examples:

```text
TTL expired
Destination unreachable
Fragmentation needed
```

Blocking ICMP can make debugging harder and may break some network behavior.

---

## 8. Final Takeaway

The Internet Protocol is the foundation of the Internet.

At Layer 3, everything becomes an IP packet.

The most important ideas are:

```text
IP address = Layer 3 identity
Packet = Layer 3 data unit
Router = forwards packets between networks
Subnet = logical network range
Subnet mask = determines same or different subnet
Default gateway = router used when destination is outside local subnet
TTL = prevents infinite packet loops
ICMP = sends control and diagnostic messages
```

For backend engineers, understanding IP helps explain problems such as:

- Slow database connections
- Routing delays
- Firewall issues
- Ping failure
- Traceroute results
- MTU problems
- TCP black holes
- Cross-subnet latency

A backend engineer does not need to be a network engineer, but understanding IP makes it much easier to debug real production systems.
