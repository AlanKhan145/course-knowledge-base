# 140 - Test Morning Brief Automation

## Section

Automation Blueprints Part 1

## Main Idea

Validate the morning brief workflow with thorough testing and quality assessment.

## Learning Objectives

By the end of this lecture, students should understand:

* Run the morning brief with real data.
* Assess output quality: completeness, clarity, relevance.
* Test edge cases: empty calendar, no news, missing data.
* Refine automation instructions based on test results.
* Establish a quality baseline for ongoing use.
