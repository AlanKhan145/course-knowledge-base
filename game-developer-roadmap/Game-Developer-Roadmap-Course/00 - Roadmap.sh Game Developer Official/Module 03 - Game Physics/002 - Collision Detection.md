# 002 - Collision Detection

**Module:** Module 03 - Game Physics
**Roadmap item:** 3.2
**Nhóm nội dung:** Game physics
**Thứ tự trong module:** 002
**Thời lượng gợi ý:** 40-55 phút

---

## 1. Tóm tắt
Bài này tập trung vào **Collision Detection** trong lộ trình Game Developer. Sau bài học, bạn nên hiểu khái niệm, biết nó xuất hiện ở đâu trong game thực tế và tạo được một demo hoặc ghi chú kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Hiểu **Collision Detection** trong bối cảnh rigidbody, lực, va chạm hoặc tối ưu physics.
- Biết phân biệt mô phỏng vật lý gameplay với mô phỏng vật lý chính xác tuyệt đối.
- Tạo được demo nhỏ để quan sát hành vi và chỉnh tham số.

## 4. Bài tập thực hành
- Tạo scene nhỏ với rigidbody/collider/force/friction và quan sát tham số thay đổi gameplay thế nào.
- So sánh ít nhất hai cách giải quyết va chạm hoặc tối ưu broad phase/narrow phase ở mức khái niệm.
- Ghi lại bug physics gặp phải và cách debug bằng gizmo/log/inspector.

## 5. Artifact nên tạo
- Physics prototype
- Collision checklist
- Bug/debug note

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Collision Detection** bằng ví dụ trong một game cụ thể không?
- Chủ đề này ảnh hưởng đến gameplay, engine, graphics, physics, AI hay build pipeline?
- Demo hoặc ghi chú nào từ bài này có thể đưa vào portfolio game developer?

**Collision Detection** nên được học bằng cách làm prototype, quan sát lỗi và ghi lại trade-off kỹ thuật. Hãy biến bài học thành một demo nhỏ hoặc README rõ ràng.
