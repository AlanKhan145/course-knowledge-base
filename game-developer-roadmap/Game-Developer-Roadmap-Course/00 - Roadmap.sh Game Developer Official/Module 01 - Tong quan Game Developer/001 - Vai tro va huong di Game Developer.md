# 001 - Vai trò và hướng đi Game Developer

**Module:** Module 01 - Tổng quan Game Developer
**Roadmap item:** 1.1
**Nhóm nội dung:** Game AI
**Thứ tự trong module:** 001
**Thời lượng gợi ý:** 40-55 phút

---

## 1. Tóm tắt
Bài này tập trung vào **Vai trò và hướng đi Game Developer** trong lộ trình Game Developer. Sau bài học, bạn nên hiểu khái niệm, biết nó xuất hiện ở đâu trong game thực tế và tạo được một demo hoặc ghi chú kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Dùng **Vai trò và hướng đi Game Developer** để thiết kế hành vi NPC, enemy, bot hoặc gameplay system.
- Biết chọn rule-based AI, search AI hoặc learning-based AI theo độ phức tạp game.
- Tạo được prototype nhỏ có state, pathfinding, decision hoặc adaptation.

## 3. Nội dung roadmap
### Mục tiêu

- Hiểu vai trò Game Developer, các hướng đi chính và kiến thức nền cần có.

### Nội dung

- Game Developer là gì?
- Client-side game development
- Server-side game development
- Các hướng chuyên sâu:
- Gameplay Programmer
- Engine Programmer
- Graphics Programmer
- Physics Programmer
- AI Programmer
- Multiplayer / Backend Game Developer
- Technical Artist

## 4. Bài tập thực hành
- Tạo enemy AI mini với patrol, chase, attack hoặc flee.
- Biểu diễn decision logic bằng state machine, behavior tree hoặc flowchart.
- Test ít nhất ba tình huống để xem AI có hành vi mong muốn không.

## 5. Artifact nên tạo
- AI behavior prototype
- State machine/behavior tree diagram
- Test cases

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Vai trò và hướng đi Game Developer** bằng ví dụ trong một game cụ thể không?
- Chủ đề này ảnh hưởng đến gameplay, engine, graphics, physics, AI hay build pipeline?
- Demo hoặc ghi chú nào từ bài này có thể đưa vào portfolio game developer?

## 7. Tổng kết
**Vai trò và hướng đi Game Developer** nên được học bằng cách làm prototype, quan sát lỗi và ghi lại trade-off kỹ thuật. Hãy biến bài học thành một demo nhỏ hoặc README rõ ràng.
