## 1. Tóm tắt
Bài này tập trung vào **Roadmap liên quan** trong lộ trình Game Developer. Sau bài học, bạn nên hiểu khái niệm, biết nó xuất hiện ở đâu trong game thực tế và tạo được một demo hoặc ghi chú kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Giải thích được **Roadmap liên quan** trong bối cảnh Game Developer Roadmap.
- Biết bài này liên quan đến nhánh gameplay, engine, graphics, physics, AI hoặc backend nào.
- Tạo một artifact nhỏ để đưa vào portfolio game developer.

## 3. Nội dung roadmap
- Roadmap liên quan
- Backend
- API Design
- Server Side Game Developer
- Roadmap.sh cũng liệt kê Game Developer trong nhóm role-based roadmaps và có roadmap Server Side Game Developer riêng .

## 4. Bài tập thực hành
- Tóm tắt bài học trong 5 dòng bằng lời của bạn.
- Tạo một demo hoặc ghi chú kỹ thuật nhỏ.
- Liên hệ bài học với portfolio game cuối khóa.

## 5. Artifact nên tạo
- Study note
- Prototype
- Portfolio README

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Roadmap liên quan** bằng ví dụ trong một game cụ thể không?
- Chủ đề này ảnh hưởng đến gameplay, engine, graphics, physics, AI hay build pipeline?
- Demo hoặc ghi chú nào từ bài này có thể đưa vào portfolio game developer?

## 7. Tổng kết
**Roadmap liên quan** nên được học bằng cách làm prototype, quan sát lỗi và ghi lại trade-off kỹ thuật. Hãy biến bài học thành một demo nhỏ hoặc README rõ ràng.
