# 002 - Shadow

**Module:** Module 07 - Lighting and Shadow
**Roadmap item:** 7.2
**Nhóm nội dung:** Lighting and rendering optimization
**Thứ tự trong module:** 002
**Thời lượng gợi ý:** 35-45 phút

---

## 1. Tóm tắt
Bài này tập trung vào **Shadow** trong lộ trình Game Developer. Sau bài học, bạn nên hiểu khái niệm, biết nó xuất hiện ở đâu trong game thực tế và tạo được một demo hoặc ghi chú kỹ thuật nhỏ.

## 2. Mục tiêu học tập
- Áp dụng **Shadow** để cải thiện hình ảnh hoặc FPS.
- Biết các lỗi thường gặp như shadow acne, overdraw, culling sai hoặc clipping khó hiểu.
- Tạo checklist kiểm tra lighting, shadow và visibility cho scene.

## 3. Nội dung roadmap
### Chủ đề

- Shadow Map
- 2D Shadow Map
- Cube Shadow Map
- Cascaded Shadow Map
- Stencil Shadow

### Cần học

- Shadow mapping
- Depth buffer
- Shadow acne
- Peter panning
- Soft shadow
- Cascaded shadow for large scenes

## 4. Bài tập thực hành
- Tạo scene có nhiều light/object rồi đo hoặc quan sát FPS trước/sau culling hoặc shadow setting.
- Ghi lại lỗi lighting/shadow và cách khắc phục.
- Tạo checklist tối ưu render cho scene lớn.

## 5. Artifact nên tạo
- Lighting checklist
- Performance note
- Before/after scene capture

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Shadow** bằng ví dụ trong một game cụ thể không?
- Chủ đề này ảnh hưởng đến gameplay, engine, graphics, physics, AI hay build pipeline?
- Demo hoặc ghi chú nào từ bài này có thể đưa vào portfolio game developer?

## 7. Tổng kết
**Shadow** nên được học bằng cách làm prototype, quan sát lỗi và ghi lại trade-off kỹ thuật. Hãy biến bài học thành một demo nhỏ hoặc README rõ ràng.
