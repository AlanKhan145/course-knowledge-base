## 1. Tom tat

Bai nay giai thich **Code Style** trong boi canh Android Developer Roadmap 2026. Sau bai hoc, ban nen biet topic nay nam o dau trong app Android, lien quan den UI, lifecycle, state, data, network, quality hoac release nhu the nao va co the bien thanh artifact trong portfolio.

## 2. Muc tieu hoc tap

- Giai thich duoc Code Style bang ngon ngu cua ban.
- Nhan biet topic nay anh huong den UX, do on dinh, maintainability, performance hoac release risk nao.
- Ap dung vao mot app nho bang code, diagram, test, checklist, screenshot hoac README.

## 3. Khai niem chinh

- Code Style is part of the Linting topic in the Android Developer roadmap.
- Static checks should run locally and in CI.
- The goal is consistent code and earlier feedback, not noisy rules for their own sake.
- Learn the concept by connecting API usage, lifecycle behavior, state and user impact.
- A strong Android developer can explain where this belongs in the app and how to test it.

## 4. Thuc hanh

1. Add the tool, test or debug technique to a sample app.
2. Run it locally and save the command/result.
3. Document what issue it catches and when to use it.

## 5. Bai tap

Add one automated or repeatable quality check and document what failure looks like.

## 6. Checklist hoan thanh

- [ ] Co dinh nghia ngan gon.

## 7. Ghi chu san xuat

Khi dua vao production, hay hoi: user flow nao bi anh huong, state co bi mat khi rotate/background khong, loi network/storage duoc xu ly ra sao, test nao bao ve tinh nang nay va release checklist co can cap nhat khong.
