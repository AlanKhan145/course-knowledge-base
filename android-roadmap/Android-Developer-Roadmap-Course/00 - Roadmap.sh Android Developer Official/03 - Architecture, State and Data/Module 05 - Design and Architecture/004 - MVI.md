## 3. Khai niem chinh

- MVI models screens as state plus events/intents plus reducers/effects.
- It is useful when UI flows become complex and you want predictable unidirectional updates.
- Keep the pattern lightweight; clarity matters more than ceremony.

## 4. Thuc hanh
