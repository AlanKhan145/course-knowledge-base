# Setup and A Challenge For You

## Overview
This lecture walks through the initial project setup for the Favorite Places app and presents a multi-part coding challenge. You are encouraged to build the foundational structure of the app independently before watching the solution videos. The challenge covers creating the data model, screens, provider, and basic navigation.

## Key Points
- A starter project is provided with basic dependencies already listed in `pubspec.yaml`
- The challenge is broken into 6 parts, each covered in a subsequent solution lecture
- You should set up the standard Flutter project folder structure: `models/`, `screens/`, `widgets/`, `providers/`
- The challenge tests your understanding of Riverpod, basic routing, and Flutter widget composition from previous modules

## Tips
- Attempt the challenge before watching the solutions to maximize learning retention
- Sketch out the data model for a "Place" (name, image, location) before writing any code
- Use the previous modules' Shopping List or Meals app as structural references

## Notes
Starter projects in this course typically include the `pubspec.yaml` with dependencies defined but no implementation code. You will need to run `flutter pub get` after opening the project to download packages. This setup phase is common in professional Flutter development workflows.

## Summary
This lecture presents the project setup and a six-part coding challenge to build the Favorite Places app skeleton before solution walkthroughs are provided.
