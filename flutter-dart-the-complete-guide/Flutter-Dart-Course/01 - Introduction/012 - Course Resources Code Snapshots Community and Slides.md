## Code Snapshots

Code snapshots are saved versions of the instructor’s code at different points in the course.

They are useful when:

* You fall behind
* Your code has an error
* You want to compare your solution with the instructor’s code
* You accidentally miss a step
* You want to restart from a specific section
* You want to study how a completed feature should look

Code snapshots are one of the most important resources in the course.

## How Code Snapshots Are Organized

Inside the GitHub repository, the `Code` folder maps to course sections.

Example structure:

```txt id="fl4k2e"
Code/
  02 Flutter & Dart Basics/
    01 Starting Setup/
    02 Some Later Snapshot/
    03 Another Snapshot/
```

The folders inside each section represent individual code snapshots.

For example:

```txt id="90r9lx"
Code/02 Flutter & Dart Basics/01 Starting Setup
```

could contain the starting project code for that course section.

## How To Use Code Snapshots

A recommended workflow:

```txt id="h5bws7"
1. Try to code along yourself
2. Run your app
3. If something breaks, read the error message
4. Compare your code with the correct snapshot
5. Find the difference
6. Fix your own code
7. Run the app again
```

The goal is not to blindly copy code.

The goal is to understand what went wrong and learn from the comparison.

## Code Snapshots For Debugging

When your app does not work, code snapshots can help you debug quickly.

Common issues include:

* Missing commas
* Wrong widget nesting
* Wrong file edited
* Wrong indentation in `pubspec.yaml`
* Typing mistakes
* Missing imports
* Wrong class names
* Missing dependencies

Comparing your project to the instructor’s snapshot can help you identify these problems.

Good ways to use slides:

```txt id="5t4rag"

  Use slides as a reference.
```

## Q&A Section

The course Q&A section is another important support resource.

Students can use it to:

* Ask questions
* Search for existing answers
* Report problems
* Find explanations from other students
* Get help when stuck

Before asking a new question, search the Q&A section first.

Many common problems have already been answered.

## How To Ask Good Questions

When asking for help, include useful details.

A good question should include:

* What you are trying to do
* What you expected to happen
* What actually happened
* The exact error message
* Relevant code
* Which lecture or section you are following
* What you already tried

This makes it easier for others to help.
