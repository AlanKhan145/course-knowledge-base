# Project Setup

## Overview
This lecture walks through setting up the Meals App Flutter project from scratch. You will create a new Flutter project, organize the folder structure, and add the initial dependencies. The lecture establishes the foundation that all subsequent lectures in this module will build upon.

## Key Points
- Create a new Flutter project using `flutter create meals_app`
- Organize folders: `lib/screens`, `lib/widgets`, `lib/models`, `lib/data`
- Clean up the default counter app boilerplate from `main.dart`
- Set up the initial `MaterialApp` and home screen scaffold
- Add any required assets or packages to `pubspec.yaml`

## Tips
- Use a consistent folder structure from the start to keep the project maintainable
- Remove all generated comments and boilerplate code for a clean slate
- Name the project carefully as it affects the package name throughout the app

## Notes
A well-organized project structure makes it easier to navigate and maintain code as the app grows. Flutter projects typically separate screens, widgets, models, and data into their own directories. Starting clean avoids confusion when adding new features later in the module.

## Summary
Setting up a clean, well-structured Flutter project is the essential first step before building any features of the Meals App.
