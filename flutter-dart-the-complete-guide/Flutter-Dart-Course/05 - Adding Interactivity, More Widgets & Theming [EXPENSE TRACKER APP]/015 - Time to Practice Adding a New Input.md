# Time to Practice Adding a New Input

## Overview
This lecture is a hands-on practice exercise where you are asked to add a new input field — specifically a dropdown or date field — to the expense form on your own. It reinforces the skills learned in previous lectures about text fields and controllers. This is a self-directed challenge before the solution is shown in the next lecture.

## Key Points
- Practice adding a new `TextField` or input widget to the `NewExpense` form
- Apply what you know about `TextEditingController` and `InputDecoration`
- Think about what additional data the expense needs (e.g., amount if not already added)
- Try to implement validation logic for the new field

## Tips
- Pause the video and attempt the exercise before watching the solution
- Refer to the Flutter documentation for any widget APIs you are unsure about
- Test your implementation by running the app and filling in the form
- Consider edge cases: empty input, invalid numbers, missing selections

## Notes
Practice exercises like this are critical for cementing newly learned concepts. Struggling through an exercise on your own — even if imperfectly — builds much stronger understanding than passively watching solutions. If you get stuck, note exactly where, then watch the solution lecture to resolve your confusion.

## Summary
This practice lecture challenges you to independently add a new input to the expense form, reinforcing text input and controller concepts from previous lectures.
