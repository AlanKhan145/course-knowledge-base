# 001 - Thuật ngữ mạng cơ bản

**Module:** Module 03 - Basic Terminology  
**Roadmap item:** Bài 4  
**Loại nội dung:** Basic terminology  
**Thời lượng gợi ý:** 35-55 phút

---

## 1. Tóm tắt
Client

Với Network Engineer, **Thuật ngữ mạng cơ bản** cần được gắn với topology, packet flow, security, reliability và khả năng troubleshooting. Mục tiêu không chỉ là thuộc khái niệm, mà là dùng nó để thiết kế hoặc xử lý sự cố mạng tốt hơn.

## 2. Mục tiêu học tập
- Giải thích được **Thuật ngữ mạng cơ bản** bằng ngôn ngữ rõ ràng cho engineer, IT support hoặc stakeholder kỹ thuật.
- Biết khi nào chủ đề này xuất hiện trong thiết kế, cấu hình, vận hành hoặc debug mạng.
- Tạo được một artifact nhỏ: sơ đồ mạng, bảng so sánh, checklist, lab note hoặc troubleshooting report.

## 3. Nội dung roadmap
- Client
- Server
- Host
- Frame
- Packet
- Latency
- Bandwidth
- Throughput
- Protocol
- Socket
- Port
- IP Address
- MAC Address
- ARP
- Transmission Media Types

## 4. Ứng dụng trong công việc Network Engineer
- Luôn xác định luồng traffic: source, destination, protocol, port, route và thiết bị trung gian.
- Ghi rõ giả định về latency, bandwidth, reliability, security và failure mode.
- Khi troubleshooting, khoanh vùng theo layer/model thay vì đoán lỗi ngẫu nhiên.

## 5. Bài tập thực hành
Tạo glossary 20 thuật ngữ mạng kèm ví dụ thực tế cho từng thuật ngữ.

Sau đó viết 5-7 dòng trả lời: nếu mạng gặp lỗi, chủ đề này giúp bạn kiểm tra điều gì trước?

## 6. Artifact nên tạo
- Networking glossary with packet/frame/IP/MAC examples
- Một sơ đồ hoặc bảng kiểm có thể dùng lại trong lab
- Một ví dụ troubleshooting: symptom, hypothesis, command/tool, result, next step

## 7. Câu hỏi tự kiểm tra
- Chủ đề này nằm ở tầng/lớp nào trong OSI hoặc TCP/IP?
- Nó ảnh hưởng tới connectivity, performance, security hay availability?
- Command, tool hoặc observation nào giúp kiểm chứng hiểu biết của tôi?
- Nếu giải thích cho người mới, tôi sẽ dùng ví dụ topology nào?

## 8. Tổng kết
**Thuật ngữ mạng cơ bản** là một phần trong năng lực thiết kế và vận hành mạng. Hãy biến bài học thành lab note, sơ đồ, checklist hoặc báo cáo troubleshooting có thể đưa vào portfolio Network Engineer.
