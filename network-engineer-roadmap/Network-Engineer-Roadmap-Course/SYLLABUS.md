# Network Engineer Roadmap - Syllabus

## Mục tiêu khóa học

Sau khóa học này, bạn có thể đọc và thiết kế topology mạng cơ bản, hiểu protocol, IP addressing, subnetting, routing, switching, wireless, load balancing, QoS, security và observability. Trọng tâm là biến khái niệm mạng thành sơ đồ, lab note, checklist và troubleshooting report có thể review.

## Lộ trình gợi ý

### Tuần 01 - Introduction

- Số bài/resource: 1
- Outcome: Hiểu Internet hoạt động ở mức tổng quan: packet, DNS, IP, TCP/UDP và HTTP/HTTPS.
- Output: Internet flow diagram and protocol overview note

### Tuần 02 - What are Networks?

- Số bài/resource: 2
- Outcome: Phân biệt các loại mạng và kiến trúc client-server, peer-to-peer.
- Output: Network type comparison table and sample topology

### Tuần 03 - Basic Terminology

- Số bài/resource: 1
- Outcome: Nắm thuật ngữ mạng nền tảng để đọc tài liệu, debug và giao tiếp kỹ thuật.
- Output: Networking glossary with packet/frame/IP/MAC examples

### Tuần 04 - Core Protocols

- Số bài/resource: 1
- Outcome: Hiểu vai trò của các protocol phổ biến như HTTP/HTTPS, TLS, SSH, DHCP, SMTP/IMAP và NTP.
- Output: Protocol cheat sheet and port/service map

### Tuần 05 - Network Devices

- Số bài/resource: 2
- Outcome: Biết chức năng của router, switch, hub, modem và access point trong topology mạng.
- Output: Network device topology diagram

### Tuần 06 - OSI Model

- Số bài/resource: 1
- Outcome: Hiểu 7 tầng OSI và dùng mô hình này để khoanh vùng lỗi mạng.
- Output: OSI troubleshooting checklist

### Tuần 07 - TCP/IP Model

- Số bài/resource: 2
- Outcome: Hiểu TCP/IP model và so sánh được với OSI trong triển khai thực tế.
- Output: OSI vs TCP/IP comparison matrix

### Tuần 08 - IP Addressing

- Số bài/resource: 2
- Outcome: Nắm IPv4, IPv6, public/private address, IP, MAC và ARP.
- Output: IP addressing worksheet and ARP flow diagram

### Tuần 09 - Subnetting

- Số bài/resource: 4
- Outcome: Biết subnet mask, CIDR, VLSM và supernetting để thiết kế địa chỉ mạng.
- Output: Subnetting workbook with CIDR/VLSM exercises

### Tuần 10 - Routing

- Số bài/resource: 3
- Outcome: Hiểu static/dynamic routing, gateway và các routing protocol chính.
- Output: Routing table examples and protocol comparison note

### Tuần 11 - Switching

- Số bài/resource: 4
- Outcome: Hiểu VLAN, STP, link aggregation và MAC address table trong switching.
- Output: Switching lab plan with VLAN/STP/MAC table observations

### Tuần 12 - DNS

- Số bài/resource: 2
- Outcome: Biết DNS hoạt động thế nào và các resolver phổ biến.
- Output: DNS lookup flow and resolver comparison note

### Tuần 13 - Wireless Networking

- Số bài/resource: 2
- Outcome: Hiểu Wi-Fi, wireless security, access point, Bluetooth, hotspot và mobile networks.
- Output: Wireless network design and security checklist

### Tuần 14 - Load Balancer

- Số bài/resource: 2
- Outcome: Hiểu load balancing, failover và thuật toán phân phối traffic cơ bản.
- Output: Load balancer decision note and failover test plan

### Tuần 15 - QoS - Quality of Service

- Số bài/resource: 2
- Outcome: Biết khi nào dùng QoS để ưu tiên traffic quan trọng và giảm nghẽn mạng.
- Output: QoS policy sketch and traffic prioritization matrix

### Tuần 16 - Network Security

- Số bài/resource: 6
- Outcome: Nắm nền tảng network security: firewall, VPN, attacks, IDS/IPS và Zero Trust.
- Output: Network security baseline and incident scenario notes

### Tuần 17 - Network Observability

- Số bài/resource: 2
- Outcome: Biết quan sát mạng bằng traffic monitoring, troubleshooting, performance và công cụ như Wireshark/Nmap/SNMP.
- Output: Observability toolkit and packet-capture troubleshooting report

### Tuần 18 - Related Roadmaps

- Số bài/resource: 1
- Outcome: Biết các roadmap liên quan để mở rộng sang DevOps, System Design, Bash, Cybersecurity và Backend.
- Output: Personal next-roadmap learning plan

## Cách học

- Với mỗi bài, tạo một artifact nhỏ: topology diagram, subnet worksheet, protocol map, routing table, security baseline hoặc troubleshooting note.
- Sau mỗi nhóm module, nối artifact vào một lab portfolio.
- Cuối khóa, hoàn thành ít nhất một lab trong `CAPSTONE_PROJECTS.md`.
