## Key Points
* Express adds a small routing and middleware layer on top of Node's HTTP server.
* It does not hide HTTP fundamentals; it organizes them into a more productive API.
* A clear Express app separates setup, routes, controllers, and error handling.
* TypeScript catches many shape and usage mistakes before the app runs.
* Express code often needs explicit types around requests, responses, and route data.
