# Technical Writer Roadmap — Dữ liệu khóa học

## Module 1: Introduction — Nhập môn Technical Writing

### 1.1 Who is a Technical Writer?

* Technical Writer là người chuyển đổi thông tin kỹ thuật thành nội dung dễ hiểu.
* Là cầu nối giữa đội kỹ thuật, sản phẩm, marketing, support và người dùng.
* Đối tượng đọc có thể là developer, end-user, buyer, support team hoặc stakeholder.

### 1.2 What is Technical Writing?

* Viết tài liệu kỹ thuật rõ ràng, chính xác, có cấu trúc.
* Nội dung thường gồm: tài liệu sản phẩm, hướng dẫn sử dụng, API docs, troubleshooting, release notes, tutorials, whitepapers.
* Mục tiêu chính: giúp người đọc hiểu, làm được việc, hoặc ra quyết định.

### 1.3 Role of Technical Writers in Organizations

* Viết và duy trì tài liệu sản phẩm.
* Làm việc với developer, product manager, designer, QA, support.
* Chuẩn hóa cách viết, style guide, thông tin sản phẩm.
* Giảm gánh nặng support bằng tài liệu tự phục vụ.

### 1.4 Forms of Technical Writing

* Product documentation
* Developer documentation
* API reference
* Help center content
* Tutorials / How-to guides
* Release notes
* Technical blog posts
* Whitepapers / eBooks
* Case studies
* Technical website copy

### 1.5 Growth as a Technical Writer

* Junior Technical Writer
* Technical Writer
* Senior Technical Writer
* Documentation Engineer
* API Documentation Specialist
* Content Strategist
* Developer Educator
* DevRel / Technical Content Marketer

## Module 2: Required Skills — Kỹ năng cần có

### 2.1 Technology Expertise

* Hiểu sản phẩm kỹ thuật, phần mềm, API, cloud, database hoặc lĩnh vực đang viết.
* Không nhất thiết phải là developer giỏi, nhưng cần đọc hiểu logic kỹ thuật.
* Biết cách hỏi developer đúng trọng tâm.

### 2.2 Language Proficiency

* Viết đúng ngữ pháp, rõ ý, tránh mơ hồ.
* Biết dùng ngôn ngữ phù hợp với trình độ người đọc.
* Có khả năng viết tiếng Anh kỹ thuật nếu làm tài liệu quốc tế.

### 2.3 Written Communication Proficiency

* Viết có cấu trúc.
* Diễn đạt ngắn gọn, chính xác.
* Biết giải thích khái niệm phức tạp bằng ví dụ đơn giản.
* Biết chỉnh sửa, rút gọn và tái cấu trúc nội dung.

## Module 4: Best Practices — Thực hành tốt khi viết

### 4.1 Storytelling

* Dùng bối cảnh thực tế để dẫn vào vấn đề.
* Giải thích vì sao người đọc cần biết nội dung này.
* Tạo mạch đọc: vấn đề → giải pháp → ví dụ → kết quả.

### 4.2 Subtle Selling

* Bán hàng nhẹ nhàng thông qua giá trị kỹ thuật.
* Không biến technical content thành quảng cáo lộ liễu.
* Tập trung vào pain point, use case và kết quả.

### 4.3 Content Structure

* Tiêu đề rõ ràng.
* Mở bài nêu mục tiêu.
* Nội dung chia section nhỏ.
* Có ví dụ, cảnh báo, lưu ý.
* Kết luận hoặc bước tiếp theo.

### 4.4 Call to Actions

* Đăng ký dùng thử.
* Đọc tài liệu tiếp theo.
* Xem API reference.
* Tải whitepaper.
* Liên hệ sales/support.
* Tham gia cộng đồng.

### 4.5 References

* Dẫn nguồn rõ ràng.
* Tránh copy nội dung không ghi nguồn.
* Ưu tiên nguồn chính thức, docs, RFC, spec, changelog.

### 4.6 Crafting Great Titles

* Tiêu đề phải nói rõ giá trị.
* Nên có keyword chính.
* Tránh tiêu đề quá mơ hồ.
* Ví dụ tốt: “How to Build REST API Documentation for a SaaS Product”.

### 4.7 Content Objectives & Intent

* Xác định mục tiêu trước khi viết.
* Người đọc muốn học, sửa lỗi, so sánh, mua hàng hay triển khai?
* Nội dung phải khớp search intent.

### 4.8 User Persona

* End-user
* Developer
* Admin
* Product manager
* CTO
* Buyer
* Support agent

### 4.9 Writing Style Guides

* Google Developer Documentation Style Guide
* Microsoft Writing Style Guide

## Module 8: Content SEO

### 8.1 Backlinking

* Xây dựng liên kết từ website khác.
* Tăng độ tin cậy cho nội dung.
* Có thể đến từ guest post, case study, tài liệu tham khảo, cộng đồng.

### 8.2 SEO Keywords

* Keyword chính.
* Keyword phụ.
* Semantic keywords.
* Search intent.

### 8.3 Short-tail Keywords

* Từ khóa ngắn, volume cao.
* Ví dụ: “API docs”, “technical writing”.

### 8.4 Long-tail Keywords

* Từ khóa dài, cụ thể hơn.
* Ví dụ: “how to write API documentation for beginners”.

## Module 11: Content Optimization

### 11.1 Tối ưu nội dung

* Cập nhật keyword.
* Cải thiện heading.
* Thêm ví dụ.
* Tối ưu CTA.
* Cập nhật internal links.
* Bổ sung schema hoặc OpenGraph nếu cần.

### 11.2 Tối ưu trải nghiệm đọc

* Đoạn văn ngắn.
* Có mục lục.
* Có code sample.
* Có hình ảnh minh họa.
* Có warning/note/tip block.

## Module 12: Content Distribution

### 12.1 Canonical Link

* Tránh duplicate content.
* Chỉ định URL chính thức cho công cụ tìm kiếm.

### 12.2 OpenGraph Data

* Tối ưu hiển thị khi chia sẻ lên mạng xã hội.
* Gồm title, description, image, URL.

### 12.3 Content Distribution Channels

* Company blog
* Newsletter
* LinkedIn
* Twitter/X
* Dev.to
* Hashnode
* Reddit
* Hacker News
* Product community
* Discord / Slack

### 12.4 Content Distribution Amplification Strategies

* Repurpose bài viết thành thread, infographic, video ngắn.
* Chia sẻ vào cộng đồng đúng ngữ cảnh.
* Dùng internal team để seed nội dung.
* Cập nhật lại bài cũ và phân phối lại.
