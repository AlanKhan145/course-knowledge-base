# Portfolio nên có

Danh sách artifact giúp chứng minh năng lực Technical Writer qua sản phẩm cụ thể.

1. Một bài **How-to Guide**
2. Một bài **API Documentation**
3. Một bài **Troubleshooting Guide**
4. Một bài **Technical Blog Post**
5. Một bài **Release Notes / Product Announcement**
6. Một trang **FAQ**
7. Một mini docs site bằng **Markdown + GitHub Pages / Docusaurus / MkDocs**

## Cách dùng portfolio

- Chọn sản phẩm hoặc API mẫu đủ đơn giản để viết nhưng đủ thực tế để review.
- Mỗi artifact nên có context, audience, objective, source notes và bản final.
- Đưa các mẫu tốt nhất vào một mini docs site bằng Markdown, Docusaurus hoặc MkDocs.
- Ghi rõ phần nào là giả lập, phần nào dựa trên sản phẩm thật hoặc open-source project.
