# 003 - User Goals

**Module:** Module 06 - Types of Technical Content — Các loại nội dung kỹ thuật
**Roadmap item:** 6.2.1
**Nhóm nội dung:** Developer Docs
**Mã nhóm:** 6.2
**Loại bài:** Developer documentation
**Thứ tự trong module:** 003
**Thời lượng gợi ý:** 35-50 phút

---

## 1. Tóm tắt
Bài này tập trung vào **User Goals** trong lộ trình Technical Writer. Sau bài học, bạn nên hiểu khái niệm, biết khi nào dùng nó và tạo được một artifact nhỏ để đưa vào portfolio.

## 2. Mục tiêu học tập
- Áp dụng **User Goals** để phục vụ developer đang cài đặt, tích hợp, debug hoặc triển khai.
- Biết cấu trúc docs theo journey và mức độ kỹ thuật của người đọc.
- Tạo mẫu API/reference/tutorial có ví dụ cụ thể.

## 3. Nội dung roadmap
* Developer muốn tích hợp API?
* Muốn cài SDK?
* Muốn debug lỗi?
* Muốn triển khai production?

## 4. Bài tập thực hành
- Viết một quickstart hoặc API reference ngắn cho REST API mẫu.
- Thêm authentication, request/response example, error code và troubleshooting note.
- Nhờ một người đọc thử và ghi lại điểm họ bị kẹt.

## 5. Artifact nên tạo
- Quickstart
- API reference sample
- Developer journey map

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **User Goals** cho một người mới vào nghề không?
- Người đọc của tài liệu này là ai và họ muốn hoàn thành việc gì?
- Artifact nào từ bài này có thể đưa vào portfolio Technical Writer?

## 7. Tổng kết
**User Goals** nên được học bằng cách viết thật, review thật và cải thiện thật. Hãy biến bài học thành một mẫu tài liệu, checklist hoặc content brief có thể tái sử dụng.
