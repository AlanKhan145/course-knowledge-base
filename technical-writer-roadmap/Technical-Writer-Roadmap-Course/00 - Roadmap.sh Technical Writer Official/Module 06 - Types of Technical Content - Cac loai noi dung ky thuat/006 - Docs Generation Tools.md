## 1. Tóm tắt
Bài này tập trung vào **Docs Generation Tools** trong lộ trình Technical Writer. Sau bài học, bạn nên hiểu khái niệm, biết khi nào dùng nó và tạo được một artifact nhỏ để đưa vào portfolio.

## 2. Mục tiêu học tập
- Hiểu vai trò của **Docs Generation Tools** trong quy trình docs-as-code hoặc xuất bản tài liệu.
- Biết khi nào dùng công cụ này trong môi trường cá nhân hoặc team.
- Tạo artifact nhỏ như repo docs, markdown page, style check hoặc publishing note.

## 3. Nội dung roadmap
* Swagger / OpenAPI
* Redoc
* Stoplight
* Postman Docs
* TypeDoc
* JSDoc
* Sphinx

## 4. Bài tập thực hành
- Tạo một file Markdown nhỏ có heading, list, table, code block và link.
- Đưa file vào Git, viết commit message rõ ràng và mô phỏng pull request review.
- Ghi lại công cụ nào phù hợp cho docs cá nhân, team nhỏ và product docs public.

## 5. Artifact nên tạo
- Markdown page
- Docs repo
- Publishing tool comparison

## 6. Câu hỏi tự kiểm tra
- Tôi có thể giải thích **Docs Generation Tools** cho một người mới vào nghề không?
- Người đọc của tài liệu này là ai và họ muốn hoàn thành việc gì?
- Artifact nào từ bài này có thể đưa vào portfolio Technical Writer?

## 7. Tổng kết
**Docs Generation Tools** nên được học bằng cách viết thật, review thật và cải thiện thật. Hãy biến bài học thành một mẫu tài liệu, checklist hoặc content brief có thể tái sử dụng.
