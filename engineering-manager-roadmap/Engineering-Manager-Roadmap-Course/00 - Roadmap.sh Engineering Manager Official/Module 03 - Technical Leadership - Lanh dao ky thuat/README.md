# Module 03 - Technical Leadership - Lãnh đạo kỹ thuật

## Mục tiêu module
Biết định hướng kỹ thuật, lập roadmap, ra quyết định kiến trúc và quản lý technical risk.

## Nội dung
- [001 - Technical Strategy](001 - Technical Strategy.md)
- [002 - Technical Roadmapping](002 - Technical Roadmapping.md)
- [003 - Architectural Decision-Making](003 - Architectural Decision-Making.md)
- [004 - Build vs Buy Evaluation](004 - Build vs Buy Evaluation.md)
- [005 - Technical Risk Assessment](005 - Technical Risk Assessment.md)
- [006 - Scaling Infrastructure](006 - Scaling Infrastructure.md)
- [007 - Legacy System Retirement](007 - Legacy System Retirement.md)

## Thực hành trọng tâm
Viết một technical strategy hoặc ADR cho một quyết định kiến trúc có nhiều trade-off.

## Output sau module
- Technical strategy memo, roadmap or ADR
- Một decision note hoặc checklist có thể dùng lại
- Danh sách rủi ro, câu hỏi mở hoặc việc cần follow-up

## Checklist hoàn thành
- [ ] Đọc toàn bộ bài học/project trong module
- [ ] Tạo ít nhất một artifact thực hành
- [ ] Liên hệ nội dung với một team, hệ thống hoặc dự án thật
- [ ] Ghi lại 3 điều sẽ áp dụng trong vai trò Engineering Manager
