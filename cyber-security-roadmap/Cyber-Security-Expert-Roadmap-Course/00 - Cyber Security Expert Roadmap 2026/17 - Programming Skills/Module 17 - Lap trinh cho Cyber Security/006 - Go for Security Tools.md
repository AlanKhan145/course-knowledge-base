# 006 - Go for Security Tools

**Học phần:** 17 - Programming Skills
**Module:** Module 17 - Lập trình cho Cyber Security
**Loại bài:** Security programming
**Thứ tự trong module:** 006
**Thời lượng gợi ý:** 25-35 phút

---

## 1. Tóm tắt
Bài này tập trung vào **Go for Security Tools** trong lộ trình **Cyber Security Expert Roadmap 2026**. Sau bài học, bạn nên hiểu khái niệm chính, biết nó liên quan đến rủi ro nào và có một artifact nhỏ để chứng minh đã thực hành.

## 2. Mục tiêu học tập
- Giải thích được **Go for Security Tools** bằng ngôn ngữ của bạn.
- Nhận biết tình huống thực tế nơi kiến thức này xuất hiện trong Cyber Security.
- Tạo được một ghi chú, checklist, lab nhỏ, báo cáo hoặc artifact phục vụ portfolio.

## 3. Nội dung chính
- CLI tools
- Networking
- Concurrency
- Lightweight scanners

## 4. Ví dụ / Lab gợi ý
```text
Input: một khái niệm, rủi ro, log, command hoặc cấu hình nhỏ
Process: áp dụng vào lab hợp pháp, ghi lại quan sát và bằng chứng
Output: ghi chú, checklist, report, script, diagram hoặc runbook ngắn
```

## 5. Bài tập thực hành
- Viết script nhỏ để parse log, kiểm tra hash, gọi API hoặc tự động hóa tác vụ an toàn.
- Thêm README giải thích input, output, cách chạy và giới hạn của script.
- Kiểm thử với dữ liệu mẫu, không dùng secret thật.

## 6. Lỗi thường gặp
- Chạy theo công cụ trước khi hiểu vấn đề nền tảng.
- Học thuộc định nghĩa nhưng không tạo artifact thực hành.
- Không ghi lại scope, giả định, bằng chứng và câu hỏi còn mở.

## 7. Checklist hoàn thành
- Tôi có thể giải thích **Go for Security Tools** trong 1-2 phút.
- Tôi có ghi chú hoặc artifact thực hành cho bài này.
- Tôi biết bài này liên quan đến risk, control, evidence hoặc owner nào.
- Tôi đã ghi lại câu hỏi còn mở để quay lại sau.

## 8. Tổng kết
**Go for Security Tools** là một mốc trong lộ trình Cyber Security. Hãy gắn nó với một ví dụ thực tế, một lab hợp pháp và một artifact nhỏ để kiến thức có điểm bám.
