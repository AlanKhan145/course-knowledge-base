# 006 - Infrastructure as Code

**Học phần:** 16 - Cloud Skills and Knowledge
**Module:** Module 16 - Cloud Security Foundations
**Loại bài:** Cloud security
**Thứ tự trong module:** 006
**Thời lượng gợi ý:** 45-60 phút

---

## 1. Tóm tắt
Bài này tập trung vào **Infrastructure as Code** trong lộ trình **Cyber Security Expert Roadmap 2026**. Sau bài học, bạn nên hiểu khái niệm chính, biết nó liên quan đến rủi ro nào và có một artifact nhỏ để chứng minh đã thực hành.

## 2. Mục tiêu học tập
- Giải thích được **Infrastructure as Code** bằng ngôn ngữ của bạn.
- Nhận biết tình huống thực tế nơi kiến thức này xuất hiện trong Cyber Security.
- Tạo được một ghi chú, checklist, lab nhỏ, báo cáo hoặc artifact phục vụ portfolio.

## 3. Nội dung chính
- Terraform concept
- CloudFormation concept
- Version-controlled infrastructure
- Security scanning IaC

## 4. Ví dụ / Lab gợi ý
```text
Input: một khái niệm, rủi ro, log, command hoặc cấu hình nhỏ
Process: áp dụng vào lab hợp pháp, ghi lại quan sát và bằng chứng
Output: ghi chú, checklist, report, script, diagram hoặc runbook ngắn
```

## 5. Bài tập thực hành
- Tạo checklist kiểm tra IAM, network exposure, logging và storage permission.
- Ghi lại risk matrix gồm likelihood, impact và remediation owner.
- Không dùng dữ liệu thật hoặc tài khoản production khi chưa có phê duyệt.

## 6. Lỗi thường gặp
- Quên shared responsibility model và quyền truy cập IAM tối thiểu.
- Học thuộc định nghĩa nhưng không tạo artifact thực hành.
- Không ghi lại scope, giả định, bằng chứng và câu hỏi còn mở.

## 7. Checklist hoàn thành
- Tôi có thể giải thích **Infrastructure as Code** trong 1-2 phút.
- Tôi có ghi chú hoặc artifact thực hành cho bài này.
- Tôi biết bài này liên quan đến risk, control, evidence hoặc owner nào.
- Tôi đã ghi lại câu hỏi còn mở để quay lại sau.

## 8. Tổng kết
**Infrastructure as Code** là một mốc trong lộ trình Cyber Security. Hãy gắn nó với một ví dụ thực tế, một lab hợp pháp và một artifact nhỏ để kiến thức có điểm bám.
