## Thực hành trọng tâm
Chọn một developer tool/API thật và vẽ developer journey từ awareness tới advocacy.

## Output sau module
- DevRel role map, developer journey map and DX audit note
- Một decision note hoặc content/support/community plan có thể chia sẻ với team
- Danh sách friction, insight hoặc feedback cần gửi lại product/engineering
